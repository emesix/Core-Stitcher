## Phase 2 — Routing policy models

Add deterministic routing as specified.

Required outcomes:

- create `orchestration/routing.py` with:
  - `RoutingPolicy`
  - `RoutingRule`
  - `EscalationTrigger`
- keep routing separate from budget logic (separate file, not inside `budget.py`)
- make precedence explicit:
  1. tag-based rules (always win)
  2. step-kind rules
  3. global default
- add `COMPUTE_TASK` to `StepKind` enum
- add `tags: list[str] = []` to `WorkRequest`, `PlannedTask`, and `TaskRecord`
- extend `ExecutorSelection` with: `matched_rule`, `dispatch_type`, `effective_tags`

Important — fallback and escalation are different concepts:

- **fallback** = same capability, different provider/instance (GPU down, try CPU on same node)
- **escalation** = result quality problem, go to stronger model (REJECT verdict, try OpenRouter)

Validate. Commit if phase is complete.

Commit message: `feat: add deterministic routing policy and routing metadata`

# VOS-Workbench — Steal Map

**Status:** Aligned — strategy still valid. Specific implementations
now defined in `alpha-proposals.md`.

*For every gap, ask: is this a standard? Adopt it. Already solved by a
mature project? Wrap it. Uniquely VOS? Define the thinnest contract possible.*

**Rule:** Do not build a custom message broker, schema language, ORM, secret
manager, UI toolkit, plugin installer, RPC protocol, or tracing system.

---

## Filter for every decision

| Category | Action | Custom code |
|----------|--------|-------------|
| **Adopt** | Clean standard exists — use it directly | None |
| **Wrap** | Project solved 80% — wrap behind VOS contract | Thin adapter |
| **Define** | Truly VOS-specific — write a minimal spec | Small contract |

---

## Gap-by-gap steal map

### A. Module type registry — Adopt + thin contract

| What | Source |
|------|--------|
| Plugin discovery | Python `importlib.metadata` entry points |
| Typed config | Pydantic config schema per module type |
| Capability patterns | OpenHands agent framework, Goose extensions |

**VOS-only contract:** Module type manifest with `type`, `entry_point`,
`config_model`, `capabilities`, `version`. Nothing more.

### B. Bootstrap sequence — Adopt

| What | Source |
|------|--------|
| Dependency resolution | `graphlib.TopologicalSorter` (stdlib 3.9+) |
| Lifecycle states | Standard service patterns, systemd-like ordering |
| Timeout + degraded mode | DAG startup patterns |

**VOS-only contract:** Circular dependency behavior, degrade vs hard-fail
rules, restart policy class.

### C. Event bus — Adopt + thin contract

| What | Source |
|------|--------|
| Phase 1: in-process pubsub | `asyncio.Queue` per subscriber |
| Durable event log | SQLite append-only table |
| Event envelope | CloudEvents spec |
| Evented runtime pattern | OpenHands event stream |

**VOS-only contract:** Event shape, correlation ID format, project scope,
replay rules, retention classes. Do not build NATS-lite.

### D. Frontend API — Adopt

| What | Source |
|------|--------|
| REST API | FastAPI |
| Live streaming | WebSocket |
| API docs | OpenAPI (auto-generated by FastAPI) |
| Reference implementation | OpenHands `openhands/server/` |

**VOS-only contract:** Endpoint design around project tree, runtime tree,
tasks, events, module health, config diffs.

### E. Config validation and settings — Adopt + define merge rules

| What | Source |
|------|--------|
| Schema validation | Pydantic v2 |
| Layered settings | `pydantic-settings` or Dynaconf |
| YAML on disk | Standard YAML |

**VOS-only contract (cannot be fully stolen):**

- Precedence: managed > bootstrap > project > local > runtime
- Scalars: replace
- Dicts: deep merge
- Lists: replace by default, keyed merge only where explicitly designed
- `null` means remove
- Hot-reload: project and below, not bootstrap

### F. Secret handling — Adopt

| What | Source |
|------|--------|
| System keyring | `keyring` library (same as Goose) |
| Env fallback | Standard env vars |
| Password store | `pass` / `gopass` |
| Vault | HashiCorp Vault client (only if needed) |

**VOS-only contract:** `secret://` URI grammar, resolution timing (lazy),
caching rule, no-plaintext export, failure behavior.

### G. Runtime storage — Adopt

| What | Source |
|------|--------|
| Database | SQLite (→ Postgres later) |
| ORM | SQLModel (SQLAlchemy + Pydantic) |
| Migrations | Alembic |
| Pattern | Standard repository pattern |

**VOS-only contract:** Table definitions for sessions, tasks, workers,
event index, module health, reconciler state, memory metadata. Artifacts
stay on filesystem.

### H. Policy engine — Wrap Claude Code pattern

| What | Source |
|------|--------|
| Rule model | Claude Code deny → ask → allow |
| Match syntax | Glob patterns (path, module type) |
| Ordering | Firewall-style priority + first-match-wins |

**VOS-only contract:** Match fields, precedence, inheritance across settings
layers, which actions are policy-aware. Start with `allow`, `ask`, `deny`.
Add `warn`, `audit`, conditions later. Do not jump to OPA/Rego.

### I. Module vs resource vs node — Define (VOS-specific)

This cannot be stolen. It is the VOS ontology.

| Concept | Definition | Examples |
|---------|-----------|----------|
| **Resource** | External thing VOS connects to | Proxmox, OPNsense, shell, Git, API provider |
| **Module** | Internal running logic inside VOS | Router, policy engine, SSH executor, memory engine |
| **Node** | Projection object for navigation/UI/runtime | Task, artifact, session, log, module view, resource view |

- Resources are external
- Modules are internal
- Nodes are presentation/runtime identity objects

### J. Concurrency — Adopt

| What | Source |
|------|--------|
| Async runtime | `asyncio` |
| Structured concurrency | `asyncio.TaskGroup` (3.11+) |
| Dangerous execution | Subprocess isolation |
| Reference | FastAPI model, OpenHands/open-interpreter execution separation |

**VOS-only contract:** When a worker is a coroutine vs subprocess, how
cancellation propagates, budget inheritance.

### K. Ephemeral worker budgets — Define (VOS-specific)

No standard exists for this. Budget envelope:

- Token budget
- Wall-clock budget
- Optional tool-call budget

**VOS-only contract:** Inheritance rule, splitting rule, exhaustion behavior,
whether child can request extension.

### L. Reconciler — Wrap K8s pattern

| What | Source |
|------|--------|
| Desired-state reconciliation | Kubernetes controller pattern |
| Config diff/apply | Standard diff loop |
| Dry-run/apply separation | Terraform-style plan/apply |

**VOS-only contract:** Triggers, scope, status transitions (converged →
pending → degraded → drifted → failed), conflict handling. Keep it small.

### M. Observability — Adopt

| What | Source |
|------|--------|
| Structured logging | `structlog` |
| Correlation IDs | Standard practice |
| Metrics/tracing | OpenTelemetry (later, not first) |

**VOS-only contract:** Required fields in every event/log, linkage between
task → worker → module → project, artifact linkage.

### N. URI grammar — Define (VOS-specific)

Follows RFC 3986 structure but needs VOS scheme definitions:

| Scheme | Resolves to |
|--------|------------|
| `module://name` | Module instance by name |
| `module://uuid` | Module instance by UUID |
| `resource://name` | Resource declaration |
| `secret://env/KEY` | Environment variable |
| `secret://file/path` | File on disk |
| `secret://pass/entry` | Password store entry |
| `secret://vault/path` | Vault secret |
| `system://eventbus` | System-level service |
| `capability://name` | Capability selector |

## Implementation phases

### Phase 1 — Steal the foundation

Freeze and implement with existing tools:

| Component | Tool |
|-----------|------|
| Config format | YAML + Pydantic |
| API | FastAPI + WebSocket |
| Storage | SQLite + SQLModel + Alembic |
| Runtime | asyncio |
| TUI (future) | Textual |
| Tool gateway | MCP Python SDK |
| Module registry | Python entry points |
| Logging | structlog |
| Secrets | keyring + env |

### Phase 2 — Define VOS-specific contracts

Write minimal specs for:

- `workbench.yaml` schema
- `module.yaml` schema
- URI grammar
- Precedence + merge rules
- Module / resource / node distinction
- Budget inheritance
- Desired / actual / effective state

### Phase 3 — Wrap proven agent patterns

Borrow from research corpus:

| Pattern | Source |
|---------|--------|
| Permission modes, compaction, budget | Claude Code |
| Evented runtime, sandbox | OpenHands |
| MCP-native tools | Goose |
| Repo/code-task ergonomics | aider |
| Execution surface separation | open-interpreter |

### Phase 4 — Defer the expensive stuff

Do not solve yet:

- Multi-project hosting
- Distributed runtime
- Postgres backend
- External message bus
- Desktop app
- Advanced agent swarms

---

## What is truly VOS-specific

Only these items require custom design:

1. Module / resource / node ontology
2. URI reference grammar
3. Settings precedence and merge rules
4. Ephemeral budget inheritance
5. Desired / actual / effective state model

Everything else is adopted or wrapped.

---

*This document is the implementation compass. When in doubt: adopt first,
wrap second, define last.*
